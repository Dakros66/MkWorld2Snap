Privacy

Your print files stay on this computer.

MkWorld2Snap is designed as a local desktop tool. It prepares 3MF/STL projects for Snapmaker Orca without adding analytics, cloud upload, or remote tracking.

Quick summary

The app reads only the files and folders you select, writes outputs where you choose, and keeps preferences locally.

  • No telemetry
  • Local processing
  • You choose output paths

No analytics or tracking

MkWorld2Snap does not include usage analytics, telemetry exporters, tracking pixels, cloud beacons, or remote statistics collection.

Project files

Files you choose are read by the local engine to inspect geometry, print settings, filament data, thumbnails, and slicer metadata.

Converted files are saved only when you choose a path in the save dialog or when automatic conversion writes into OUTPUT_U1 inside a watched folder.

Temporary working data

During conversion the app may create temporary files so the rebuilt package can be previewed and saved. Those files are short-lived working data, not a permanent project library.

Watched folders

Automatic conversion only scans the folders you add. It looks for supported model files and writes converted outputs to the matching OUTPUT_U1 folder.

Preferences stored locally

The app window may use browser-style localStorage to remember language, theme, and workflow preferences. It is not used to store model contents or personal information.

Start with computer

If you enable start with computer, the app creates the operating-system startup entry needed to open MkWorld2Snap after login. You can disable it from the app.

Model ownership

Use files you have the right to modify. MkWorld2Snap prepares project packages only; models, designs, licenses, and attribution remain yours or their original owner’s.

Network access

The desktop app serves its own local interface and talks to its local conversion engine. It does not need an external account to process files.
